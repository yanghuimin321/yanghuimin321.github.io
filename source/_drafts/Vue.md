- **渐进式框架**： 一种允许开发者根据项目需求**逐步采用**其功能的框架设计理念。
  - 核心思想：可以从轻量级使用开始，逐步扩展为功能完整的方案
  - 分层架构：
    - **核心库**： 只关注视图层，处理基本视图渲染/响应式数据绑定
    - **附加层**： 路由、全局状态管理、构建工具等
- 对比其他框架：







- 只有当Vue实例被创建时就已经存在于data中的property才是响应式的，因此需要提前把需要的property都设置好；使用Object.freeze()会导致响应系统无法追踪变化；
- 不要在选项 property 或回调上使用箭头函数，因为箭头函数没有this，this会作为变量一直向上级词法作用域查找；标准函数中，this引用的是把函数当成**方法调用的上下文对象**；箭头函数中，this引用的是**定义箭头函数的上下文**
- 计算属性computed基于它们的**响应式依赖**进行缓存，只有相关响应式依赖发生变化才会重新执行函数求值；方法methods每当触发**重新渲染**，都会再次执行函数。
- Vue会尽可能高效地渲染元素，通常会复用已有元素而不是从头开始渲染；通过唯一值的`key` attribute表示不要复用



| 模板语法（单个JS表达式）  | 详情                                                         |
| ------------------------- | ------------------------------------------------------------ |
| **“Mustache”语法**        | `{{}}`，文本插值                                               |
| **v-once**                | 执行一次性插值                                               |
| **v-html**                | 将数据解释为html代码填充；v-html会忽略解析 property 值中的数据绑定；不能使用 `v-html` 来复合局部模板，因为 Vue 不是基于字符串的模板引擎（慎用，容易导致XSS攻击） |
| **v-bind**(缩写：)        | 作用在HTML attribute上 :[动态参数].修饰符                    |
| **v-on**(缩写@)           | 作用在DOM事件上@[动态参数].修饰符                            |
| **v-if,v-else-if,v-else** | 可以在<template>上使用，最终会不渲染<template>；惰性的；有更高的切换开销 |
| **v-show**                | 元素始终被渲染并保留在DOM中，只是简单地切换元素的CSS property `display`；有更高的初始渲染开销；不支持<template>元素 |
| **v-for**                 | 不推荐和v-if一起使用；具有比v-if更高的优先级；               |
